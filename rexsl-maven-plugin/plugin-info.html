Plugin Documentation

Goals available for this plugin:

Goal Description
rexsl:check Test entire project against RESTful principles.
rexsl:help Display help information on rexsl-maven-plugin.
Call mvn rexsl:help -Ddetail=true -Dgoal=<goal-name> to display parameter details.
rexsl:package Package resources.

Text resources should be packed before they got delivered to the WAR package. For example, comments should be removed from CSS and XSL stylesheets, JavaScripts should be minified, etc. We have a number of convenient packers that do this job, you just need to call them. First of a package goal should be called:

<plugin>
  <groupId>com.rexsl</groupId>
  <artifactId>rexsl-maven-plugin</artifactId>
  <version>...</version>
  <executions>
    <execution>
      <phase><phase>
      <goals>
        <goal>package</goal>
      </goals>
    </execution>
  </executions>
</plugin>

The goal will package all files it knows about and place their packaged (minified) versions into webappDirectory. Then, you should configure maven-war-plugin to not touch these files again (to avoid their packaging):

<plugin>
  <artifactId>maven-war-plugin</artifactId>
  <configuration>
    <warSourceExcludes>js/**, css/**, xsl/**</warSourceExcludes>
    [..]
  </configuration>
</plugin>
rexsl:run Run WAR as a web server to enable interactive testing of a web application.

Run mvn clean package rexsl:run -Drexsl.port=9099 and open your browser at http://localhost:9099/. You will see the front page of your application. What is especially interesting is that you can make changes to static resources (xsl, css and other files in src/main/webapp) and then just refresh the page in a browser. New versions of the files will be delivered. That's how you can interactively test your XSL stylesheets.

System Requirements

The following specifies the minimum requirements to run this Maven plugin:

Maven 2.0
JDK 1.6
Memory No minimum requirement.
Disk Space No minimum requirement.

Usage

You should specify the version in your project's plugin configuration:

<project>
  ...
  <build>
    <!-- To define the plugin version in your parent POM -->
    <pluginManagement>
      <plugins>
        <plugin>
          <groupId>com.rexsl</groupId>
          <artifactId>rexsl-maven-plugin</artifactId>
          <version>0.5.1</version>
        </plugin>
        ...
      </plugins>
    </pluginManagement>
    <!-- To use the plugin goals in your POM or parent POM -->
    <plugins>
      <plugin>
        <groupId>com.rexsl</groupId>
        <artifactId>rexsl-maven-plugin</artifactId>
        <version>0.5.1</version>
      </plugin>
      ...
    </plugins>
  </build>
  ...
</project>

For more information, see "Guide to Configuring Plug-ins"
